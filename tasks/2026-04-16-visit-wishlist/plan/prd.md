# PRD — 방문 위시리스트

**작성일:** 2026-04-16
**작성자:** planner
**상태:** 확정

## 목적
기존 위험 장애물 지도 앱에 "방문 위시리스트" 기능을 추가한다.
사용자가 방문하고 싶은 장소를 마커로 저장하고, 방문 횟수를 기록하며, 방문 상태와 무관하게 후기를 남길 수 있게 한다.
장애물 관리와는 독립된 별도 데이터 레이어로 운영한다.

## 대상 사용자
기존 장애물 지도 앱 사용자. 장소 방문 계획을 세우고 후기를 기록하고 싶은 사람.

## 핵심 요구사항

### 기능 요구사항 (반드시 구현)
- 지도에서 클릭하여 위시리스트 장소를 등록할 수 있다
- 위시리스트 항목에는 장소명, 주소(선택), 카테고리를 기록한다
- 카테고리는 위시리스트 전용: 맛집, 카페, 관광지, 공원, 쇼핑, 문화/예술, 기타
- 방문 완료 버튼으로 visitCount를 증가시킬 수 있다 (0 = 미방문, 1+ = 방문완료)
- 방문 상태와 무관하게 후기(텍스트)를 여러 개 작성할 수 있다
- 후기에 선택적으로 "몇 번째 방문 후기인지"를 기록할 수 있다
- 위시리스트 전용 마커 아이콘으로 장애물 마커와 시각적으로 구분된다
- 사이드바/바텀시트에서 위시리스트 목록을 조회, 검색, 필터링할 수 있다

### 비기능 요구사항
- localStorage 기반 저장 (기존 obstacles 저장 패턴과 동일)
- 기존 장애물 기능에 영향 없음 (독립 데이터, 독립 UI 레이어)
- 모바일/데스크톱 반응형 지원 (기존 앱과 동일 수준)

## 데이터 구조

```json
{
  "id": "uuid",
  "name": "string (장소명)",
  "category": "맛집|카페|관광지|공원|쇼핑|문화/예술|기타",
  "lat": "number",
  "lng": "number",
  "address": "string (선택적)",
  "visitCount": "number (0 = 미방문)",
  "reviews": [
    {
      "id": "uuid",
      "content": "string",
      "createdAt": "ISO string",
      "visitNumber": "number | null (몇 번째 방문 후기인지, 선택적)"
    }
  ],
  "createdAt": "ISO string",
  "updatedAt": "ISO string"
}
```

## 범위

### In Scope
- 위시리스트 CRUD (생성, 조회, 수정, 삭제)
- 방문 횟수 카운트 (visitCount++ 버튼)
- 후기 작성 (방문 상태 무관, 다중 후기 가능)
- 위시리스트 전용 마커 렌더링
- 위시리스트 목록 UI (사이드바, 바텀시트)
- 장애물/위시리스트 탭 전환 UI
- 카테고리별, 방문 여부별 필터링

### Out of Scope
- 이미지 첨부 (후기에 사진 추가)
- 다른 사용자와 위시리스트 공유
- 서버 연동 / 클라우드 동기화
- 선(Line)/면(Polygon) 타입 위시리스트 (점 마커만 지원)
- 별점 시스템 (단순 텍스트 후기만)

## 성공 기준
1. 지도 클릭으로 위시리스트 장소를 등록할 수 있다
2. 등록된 장소가 지도에 별도 아이콘으로 표시된다
3. 방문 완료 처리로 visitCount가 증가하고 UI에 반영된다
4. 방문 상태와 관계없이 후기를 작성하고 목록으로 조회할 수 있다
5. 새로고침 후에도 데이터가 유지된다 (localStorage)
6. 기존 장애물 기능이 정상 동작한다
