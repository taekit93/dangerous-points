# PRD — Line/Polygon 장애물 타입 추가

**작성일:** 2026-04-16
**작성자:** planner
**상태:** 초안

## 목적
현재 점(Point) 타입만 지원하는 장애물 등록 서비스에 선(Line) 타입과 면(Polygon) 타입을 추가하여, 선형 장애물(도로 균열, 공사 구간 등)과 영역 장애물(침수 지역, 위험 구역 등)을 보다 정확하게 표현할 수 있도록 한다.

## 대상 사용자
장애물 지도 서비스의 기존 사용자. 지도에서 클릭/드래그로 장애물을 등록하는 현장 작업자 또는 관리자.

## 핵심 요구사항

### 기능 요구사항 (반드시 구현)
- 장애물 등록 시 타입 선택 UI (Point / Line / Polygon)
- Line 타입: 지도 위에서 2개 이상의 좌표를 순차 클릭하여 선형 장애물 등록
- Polygon 타입: 지도 위에서 3개 이상의 좌표를 순차 클릭하여 다각형 영역 장애물 등록
- 각 타입에 맞는 네이버 지도 오버레이 렌더링 (Polyline, Polygon)
- 기존 Point 타입과 동일한 메타데이터 입력 (제목, 카테고리, 위험도, 설명)
- 기존 필터링(카테고리, 위험도)이 새 타입에도 동일하게 동작
- localStorage 저장 구조 호환 (기존 Point 데이터 마이그레이션 불필요 — type 필드 없으면 'point'로 간주)
- 장애물 목록(MarkerList)에서 타입 아이콘/뱃지로 구분 표시

### 비기능 요구사항
- 기존 Point 타입의 동작이 변경되지 않아야 함 (하위 호환)
- 네이버 지도 API v3 Drawing 없이 기본 API만 사용 (현재 코드베이스 기준)
- 기존 테스트가 깨지지 않아야 함
- 모바일 반응형 레이아웃 유지

## 범위

### In Scope
- 장애물 데이터 모델에 type 필드 추가 ('point' | 'line' | 'polygon')
- 장애물 데이터 모델에 coords 필드 추가 (Line/Polygon용 좌표 배열)
- 지도 위 다중 좌표 입력 인터랙션 (클릭으로 좌표 추가, 완료 버튼으로 확정)
- Polyline / Polygon 오버레이 렌더링 및 색상(위험도 기반)
- ObstacleForm에 타입 선택 UI 추가
- MarkerList에 타입 표시 추가
- InfoWindow 내용에 타입 정보 표시
- Line/Polygon 오버레이 클릭 시 InfoWindow 표시
- 기존 Point 데이터 하위 호환 (type 미지정 시 point로 처리)

### Out of Scope
- 좌표 드래그 수정 (등록 후 개별 좌표 이동)
- 장애물 영역 면적/길이 자동 계산
- 서버 연동 (현재 localStorage 기반 유지)
- 네이버 지도 Drawing Manager 사용

## 성공 기준
1. Point / Line / Polygon 세 가지 타입으로 장애물을 등록할 수 있다
2. 각 타입이 지도 위에 적절한 오버레이(마커, 폴리라인, 폴리곤)로 표시된다
3. 기존에 등록된 Point 장애물이 정상 동작한다 (하위 호환)
4. 필터링, 검색, 삭제, 수정이 세 타입 모두에서 동작한다
5. 기존 테스트(vitest)가 모두 통과한다
6. 새 타입에 대한 테스트가 추가된다
