**이유:** React + Vite 기본 지원으로 추가 설치 불필요. CSS Custom Properties와 완벽 호환. 네이버지도 InfoWindow HTML 문자열과의 충돌 최소화(전역 info-window.css 분리). Tailwind는 위험도별 동적 색상 상태 표현 시 가독성 저하 우려.
**대안:** Tailwind CSS(동적 상태 처리 복잡도 증가), styled-components(런타임 오버헤드)

## 2026-04-16 09:00 — designer

**이유:** 공공/현장 서비스 성격에 맞는 신뢰감 있는 네이비(#2D3E50). 위험도 색상은 직관적인 신호등 체계(빨강/주황/초록) 적용. 보라색 그라디언트·흰 배경 조합은 AI 슬롭으로 금지 지침 준수.
**대안:** 밝은 파란색 계열(너무 일반적), 다크 모드 우선(사용자층 불명확)

## 2026-04-16 09:00 — designer

**결정:** 모바일 패널 — 하단 시트(Bottom Sheet) 패턴 채택
**이유:** 지도 서비스 특성상 지도를 최대한 노출해야 함. 하단 시트는 지도를 가리지 않고 필요 시 펼치는 UX 패턴. 3단계 높이(접힘/절반/전체)로 유연한 공간 활용.
**대안:** 좌측 드로어(지도 절반 가림), 상단 오버레이(지도 조작 방해)

## 2026-04-16 09:00 — designer

**결정:** ObstacleForm 데스크탑 — 지도 위 중앙 모달, 모바일 — 하단 시트
**이유:** 데스크탑에서는 등록 위치 좌표와 폼을 동시에 확인 가능. 모바일에서는 키보드 UX를 고려해 하단 시트 패턴이 자연스러움. iOS 자동 줌 방지를 위해 입력 font-size: 16px 명시.
**대안:** 항상 사이드바 인라인 폼(모바일에서 지도 완전 숨김)
