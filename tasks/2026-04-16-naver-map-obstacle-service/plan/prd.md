# PRD — 네이버지도 기반 장애물 등록 서비스

**작성일:** 2026-04-16
**작성자:** planner
**상태:** 확정

## 목적
네이버지도 위에 장애물(위험 지점)을 마커로 등록, 조회, 수정, 삭제할 수 있는 웹 서비스를 만든다.
백엔드 없이 localStorage만으로 데이터를 관리하여 별도 서버 배포 없이 즉시 사용 가능한 경량 도구를 제공한다.
각 장애물에는 카테고리와 위험도(위험/주의/안전)를 함께 지정하여 시각적으로 구분한다.

## 대상 사용자
- 특정 지역의 장애물/위험 지점을 기록하고 관리하려는 개인 사용자
- 현장 조사 시 위치 기반으로 장애물 정보를 수집하는 담당자

## 핵심 요구사항

### 기능 요구사항 (반드시 구현)
- 네이버지도 표시 및 기본 조작 (줌, 이동)
- 지도 클릭으로 장애물 마커 등록 (폼 팝업)
- 마커에 카테고리(자유 입력 또는 선택)와 위험도(위험/주의/안전 3단계) 모두 입력
- 위험도별 마커 색상 구분: 위험(빨강), 주의(주황), 안전(초록)
- 마커 클릭 시 장애물 상세 정보 인포윈도우 표시
- 장애물 정보 수정 기능
- 장애물 삭제 기능 (확인 다이얼로그 포함)
- 사이드바 장애물 목록 조회 (항목 클릭 시 해당 마커로 지도 이동)
- 카테고리 + 위험도 필터링
- localStorage 기반 CRUD (Create, Read, Update, Delete)
- 반응형 UI: 모바일 하단 시트, 데스크탑 좌측 사이드바

### 비기능 요구사항
- 기술 스택: React + Vite
- 네이버 Maps JavaScript API v3 (웹 Dynamic Map)
- 모바일 퍼스트 반응형 디자인
- localStorage 용량 한계 고려 (약 5MB)
- .env.example에 API 키 placeholder + 설정 안내

## 범위

### In Scope
- React + Vite 프로젝트 초기화
- 네이버지도 연동 및 마커 표시
- 장애물 CRUD (등록/조회/수정/삭제)
- 카테고리 시스템 (기본 6종 + 사용자 자유 추가)
- 위험도 3단계 (위험/주의/안전) 및 색상 마커
- localStorage 데이터 영속화
- 반응형 레이아웃 (모바일 하단 시트 ~ 데스크탑 좌측 사이드바)
- 카테고리 + 위험도 필터링
- .env.example 및 환경변수 설정 안내

### Out of Scope
- 서버 사이드 백엔드 / DB
- 사용자 인증 / 로그인
- 다중 사용자 간 데이터 공유
- 이미지 첨부 (localStorage 용량 제한)
- 오프라인 PWA 지원
- 데이터 내보내기/가져오기 (백업 기능 미포함)

## 성공 기준
- 지도 위에서 클릭하여 장애물을 등록할 수 있다
- 등록 시 카테고리와 위험도를 모두 입력할 수 있다
- 위험도에 따라 마커 색상이 다르게 표시된다 (빨강/주황/초록)
- 등록된 장애물이 localStorage에 저장되어 새로고침 후에도 유지된다
- 등록된 장애물을 목록으로 조회하고, 수정/삭제할 수 있다
- 카테고리와 위험도로 필터링이 동작한다
- 모바일(360px)과 데스크탑(1920px) 환경 모두에서 정상 동작한다
- .env에 네이버지도 API 키만 설정하면 즉시 사용 가능하다
